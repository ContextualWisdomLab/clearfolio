## 2026-06-24 - 비동기 상태에 따른 동적 문서 제목
**Learning:** 문서 변환과 같이 길어지는 비동기 폴링 작업 중에 페이지 상태가 라이브 영역에서만 업데이트될 경우, 스크린 리더 사용자는 컨텍스트를 잃기 쉽습니다. 비동기 상태가 로딩에서 오류나 성공으로 전환될 때 `document.title`을 변경하면 앱의 전역 상태가 변경되었다는 피드백을 즉각적으로 제공할 수 있습니다.
**Action:** 싱글 페이지 앱이나 비동기 통신이 잦은 뷰에서는 항상 주요 UI 상태 변경(예: `로딩 중`, `오류`, `준비됨`)을 `document.title`에 매핑해야 합니다.
