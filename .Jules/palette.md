## 2026-06-23 - External Links & Disabled Button States in Async Polling
**Learning:** For external resource links (like the "Open JSON bootstrap" or generated artifact links), lacking `target="_blank"` combined with `rel="noopener noreferrer"` can create poor UX, where users lose context of the viewer app. Furthermore, in `viewer.js`, allowing the refresh/retry button to remain enabled during the async polling causes screen readers to have a confusing experience where users can repeatedly click the button, generating duplicate API calls and multiple `aria-busy` announcements.
**Action:** Always pair `target="_blank"` with `rel="noopener noreferrer"`, use `aria-label` to explicitly state "(opens in a new tab)", and reliably toggle the `disabled` and `aria-busy` states on interactive elements during asynchronous JavaScript polling so assistive technologies receive correct context updates.
