## 2026-06-26 - External Link Accessibility and UX Pattern
**Learning:** Screen reader users and general users need clear indicators when a utility link (like an API bootstrap JSON or health check) opens in a new tab. Without these cues, it breaks the navigation mental model and causes confusion. Furthermore, grouping elements (like buttons and links) with `aria-label` but missing the explicit `role="group"` container role causes screen readers to skip announcing the label.
**Action:** When adding utility or external links, always apply the `target="_blank" rel="noopener noreferrer"` attributes, append a hidden visual indicator (`<span aria-hidden="true">&nearr;</span>`), and add a clear `aria-label` (e.g., "Link text (opens in a new tab)"). Always use `role="group"` on action wrapper divs when labeling them with `aria-label`.
