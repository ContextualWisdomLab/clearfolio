## 2026-06-28 - Clean up empty state text dynamically and use semantic roles for aria-labels
**Learning:** We must properly clean up empty state text (like `.help` messages) when content successfully loads. Otherwise, the help text remains under the loaded content, causing confusing UX. Also, `aria-label` attributes must be applied to elements with appropriate semantic roles (like `role="group"`) to be correctly interpreted by assistive technologies. Using redundant `aria-label`s on elements that already have text content causes screen readers to announce the element twice.
**Action:** When creating empty states, always ensure there is matching Javascript logic to hide or remove the state upon successful load. For accessibility, always pair `aria-label` with a valid semantic role and avoid adding `aria-label` when the element's text content is already sufficient.
