## 2026-06-25 - [Global Security Headers Implementation]
**Vulnerability:** Baseline security headers (like `X-Content-Type-Options`, `Strict-Transport-Security`, `X-XSS-Protection`, and general API `Content-Security-Policy` and `X-Frame-Options`) were missing on API routes. They were only partially applied to the HTML viewer route `/viewer/**`.
**Learning:** Security headers should be treated as a global cross-cutting concern. Even JSON API endpoints need them to prevent MIME-sniffing attacks (`nosniff`) or improper inclusion in `iframe` or `script` tags if browsers attempt to execute them. By restricting frame-ancestors globally and adding base CSP to APIs, we reduce the blast radius for clickjacking and generic injection vectors.
**Prevention:** Always implement a global WebFilter or Interceptor to handle baseline security headers (HSTS, X-Content-Type-Options, frame/ancestors protections). Treat API routes as susceptible surfaces for client-side inclusion attacks.
